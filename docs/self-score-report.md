Once the capstone lands in the public repo and a real scorer run is executed, the structured markdown output from `alexeygrigorev/github-project-scorer` overwrites this file. The scorer writes sections per criterion with:

Keep this file committed so a grader visiting the repo sees a score without needing to run the scorer themselves. It also lets reviewers compare consecutive runs over time.
